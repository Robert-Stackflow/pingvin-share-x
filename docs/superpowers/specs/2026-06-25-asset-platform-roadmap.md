# Pingvin Share X — Asset 平台重构总览路线图

> 状态:已与用户对齐拆分与顺序(2026-06-25)。本文是整个重构的锚点,
> 各子项目有独立的 spec → plan → 实现闭环。

## 背景

Pingvin Share X 当前以 `Share` 为核心实体,`Share` 内含 `File[]`。现要新增三大能力,
并借此重构架构,同时全面优化前端。

## 已确认的核心决策

| 决策点 | 选择 |
|---|---|
| Asset 与 Share 关系 | **Asset 成为顶层独立实体**(FILE/TEXT/LINK),可独立存在,也可挂到 Share / 剪贴板下 |
| 在线剪贴板模型 | **私人剪贴板 + 口令/房间式共享空间**(两者都做) |
| 短链范围 | **站内资源 + 任意外部 URL**(通用短链) |
| 短链统计深度 | **明细访问日志 + 图表**(Redis 计数/缓存) |
| 现有数据 | **无需迁移,可重置数据库** |
| 前端尺度 | **先升级 Mantine 到最新,再做全面重设计** |

## 子项目拆分与依赖

| # | 子项目 | 内容 | 依赖 |
|---|--------|------|------|
| **0** | **Mantine 8 升级** | 纯机械迁移(去 emotion / `@mantine/next` / `createStyles` → CSS Modules),不加新功能 | 无 |
| **1** | **Asset 核心模型** | 后端引入 `Asset`(FILE/TEXT/LINK)顶层实体;Share 重构为 Asset 容器;统一存储/下载/权限抽象;配套基础前端 | 0 |
| **2** | **在线剪贴板** | 私人剪贴板 + 口令房间;基于 Asset;剪贴板前端页 | 1 |
| **3** | **短链 + 分析** | 通用短码(站内/外部)、Redis 计数、明细日志、图表 dashboard | 1(站内引用) |

**开发顺序:0 → 1 → 2 → 3。**

前端"全面优化"拆进各子项目(每个新功能带自己的页面重设计),收尾再做一次全局设计系统打磨,
而非单列一个大前端项目。
